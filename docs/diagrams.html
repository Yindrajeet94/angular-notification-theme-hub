⚡ Notification Flow — Step by Step

What happens when the user clicks "✅ Success" in the playground.

1
User clicks a button in SubjectPlayground
Angular's event binding fires the click handler.
(click)="notify('Employee saved!', 'success')"
2
SubjectPlayground calls NotificationService.show()
The component itself does not know who will display the notification. It delegates entirely to the service.
this.notificationService.show(message, type)
3
NotificationService builds the Notification object and calls .next() on 3 subjects
A single show() call fans out to three independent streams simultaneously.
notificationSubject.next(n)
Plain Subject — live toast stream
historySubject.next(n)
ReplaySubject — persistent history
countSubject.next(count++)
Subject — bell badge number
4
All subscribers receive their value synchronously
RxJS Subjects push synchronously — the callbacks run inline, before .next() even returns.
ToastContainer
activeToasts.push(n)
sets setTimeout for auto-dismiss
NotificationHistory
notifications.push(n)
list re-renders with new item
Header
notificationCount = count
badge appears / increments
5
cdr.markForCheck() triggers Angular to re-render (zoneless)
Because this app has no Zone.js, each subscription callback calls this.cdr.markForCheck() after mutating state. Angular then schedules a change-detection pass and updates the DOM.
this.cdr.markForCheck() // injected ChangeDetectorRef

🌙 Theme Toggle Flow

What happens when the user clicks the dark/light toggle in the Header.

1
Header calls ThemeService.toggleTheme()
(click)="toggleTheme()" → this.themeService.toggleTheme()
2
ThemeService reads current value and pushes the opposite
Only BehaviorSubject supports .getValue() — regular Subject has no current value.
const current = this.themeSubject.getValue(); // 'light'
this.themeSubject.next(current === 'light' ? 'dark' : 'light');
3
ALL components subscribed to theme$ receive the new value instantly
Five components — App, Header, Toast, Playground, History — all update their themeConfig and call markForCheck(). Every background, text, and border color transitions smoothly.
AppComponent
app-shell background
HeaderComponent
header bar + toggle thumb
SubjectPlayground
card background + text
NotificationHistory
panel background + borders

🔄 Zoneless Change Detection Pattern

Without Zone.js, Angular doesn't know when async callbacks (RxJS subscriptions, setTimeout) mutate state. Each component manually signals Angular to re-check.

❌ With Zone.js (not this app)
subscription fires
→
Zone.js patches the callback
→
Angular notified automatically
→
DOM updates
✅ Without Zone.js (this app)
subscription fires
→
state updated in callback
→
cdr.markForCheck()
→
Angular schedules check
→
DOM updates
Applies to: RxJS subscription callbacks · setTimeout callbacks (toast auto-dismiss) · any async callback that mutates component state
